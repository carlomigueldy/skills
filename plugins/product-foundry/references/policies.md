# Shared safety, delegation, and completion policies

`find-skills` is mandatory before a product-specific roster is created.
`frontend-design` is mandatory before either prototype phase. If unavailable,
the affected phase fails closed with recovery guidance.

Delegate mechanical collection to low-cost capable agents, scoped synthesis and
implementation to balanced agents, and use stronger review only for ambiguous,
security-sensitive, or cross-domain reconciliation. Record every escalation.
Default delegation depth is two; deeper delegation requires a written
justification. Shared state has one writer, delegated documents get parent
review evidence, and the root approves cross-domain changes.

Require explicit approval before production deployment, paid services, domains,
accounts, secrets, destructive data work, legal acceptance, or irreversible
actions. An approval request states impact, cost or risk, rollback, and safer
alternatives. Completion is fail-closed until required evidence, documentation,
quality gates, and release readiness are current.
